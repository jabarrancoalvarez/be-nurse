Crea el proyecto completo "be-nurse" desde cero con el siguiente stack y requisitos. Es una plataforma de salud sexual respaldada por profesionales de enfermería que conecta pacientes con enfermeros/as especializados/as. El proyecto original estaba en Next.js pero lo migramos a Angular + .NET Core + PostgreSQL.

STACK TECNICO
- Frontend: Angular 18+ (standalone components, signals, nueva sintaxis @if/@for)
- Backend: .NET Core 8 Web API (Clean Architecture)
- Base de datos: PostgreSQL 16
- ORM: Entity Framework Core con Npgsql
- Autenticacion: JWT (preparado pero no activo en primera version)
- Docker: docker-compose para levantar todo el entorno local

FRONTEND ANGULAR 18

Genera el proyecto Angular con "ng new frontend --standalone --routing --style=scss". Todas las rutas deben usar lazy loading.

Rutas de la aplicacion:
- / → LandingComponent
- /informate → InformateComponent
- /cuidate → CuidateComponent
- /realidades → RealidadesComponent
- /realidades/:slug → RealidadDetalleComponent
- /about → AboutComponent
- /chat → ChatComponent

Archivo app.config.ts: incluye provideRouter con las rutas, provideHttpClient(), provideAnimations().

Archivo app.routes.ts: todas las rutas con loadComponent para lazy loading.

Dockerfile para el backend (en ./backend/Dockerfile):
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
EXPOSE 80
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY . .
RUN dotnet restore BeNurse.API/BeNurse.API.csproj
RUN dotnet build BeNurse.API/BeNurse.API.csproj -c Release -o /app/build
FROM build AS publish
RUN dotnet publish BeNurse.API/BeNurse.API.csproj -c Release -o /app/publish
FROM base AS final
WORKDIR /app
COPY --from=publish /app/publish .
ENTRYPOINT ["dotnet", "BeNurse.API.dll"]

README.md en la raiz con instrucciones para:
1. Requisitos previos: Node 20+, .NET SDK 8, Docker Desktop
2. Levantar con Docker: docker-compose up --build
3. Desarrollo local: cd backend && dotnet run --project BeNurse.API, cd frontend && ng serve
4. URLs: Frontend http://localhost:4200, API http://localhost:5000, Swagger http://localhost:5000/swagger
5. Migraciones: dotnet ef migrations add InitialCreate --project BeNurse.Infrastructure --startup-project BeNurse.API

Una vez generado todo el codigo, ejecuta los comandos necesarios para verificar que el frontend compila (ng build) y que el backend compila (dotnet build). Corrige cualquier error de compilacion antes de terminar.
